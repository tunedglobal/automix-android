    Configure the SDK with custom Backend¶

    Initialise with Custom backend¶

    In order to initialise AutomixIQ when you take the responsability of providing audio data and metadata you need the application context, a AutomixSDKStreamingListener, plus a store ID, applicationId and a country code.

    override fun onStart(arguments: Bundle?) {
    // ...

    val applicationContext: ApplicationContext = myApplicationContext
    val automixSDKStreamingListener: AutomixSDKStreamingListener = myAutomixSDKStreamingListener

    val countryCode: String = myCountryCode // "US" ? "AU"?
    val storeId: String = myTunedStoreId
    val applicationId: String = myApplicationId
    val config = AutomixSDKConfig.Builder().country(countryCode).storeId(storeId).applicationId(applicationId).logLevel(4).build()

    AutomixSDK.initialise(applicationContext,
                          automixSDKStreamingListener,
                          config,
                          onComplete = 
    { _ -> 
        // The Automix SDK public functions can be used from this point onwards :)

        // We can add a status listener to get UI updates and play events
        val automixSDKStatusListener: AutomixSDKStatusListener = myAutomixSDKStatusListener
        AutomixSDK.getInstance()?.addStatusListener(automixSDKStatusListener)

        // ...
    }, onError = { error, _ ->
        // This means you are fobidden to use the Automix SDK or you have no internet connection.
    })

    // ...
}

    Implementing an instance of AutomixSDKStreamingListener¶

    AutomixSDKStreamingListener provides AutomixIQ with the information from your service that it needs in order to play. The functions of this interface need to be implemented.

    • If your custom backend puts fetching metadata and track data behind a login, AutomixIQ needs to know if the user in your aspplication is logged in or not. To provide this information, implement isLoggedIn. If your custom backend does not require a login, this function should return true.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun isLoggedIn() {
        return this.loggedIn
    }
    // ...
}
    • If your custom backend only allows playback on a device under certain conditions (for instance in only allows one device to play at any given time), AutomixIQ needs to know if a device is allowed to play. To get this information, implement asyncCheckDeviceStatus and run onComplete with the response. If so such requirement exists, this function should run onComplete(true).
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun asyncCheckDeviceStatus(onComplete: ((Boolean) -> Unit)?,
                                        onError: ((AutomixSDKError) -> Unit)?) -> Unit) {
        this.canDeviceContinuePlaying() { allowedToPlay ->
            onComplete(allowedToPlay)
        }
    }
    // ...
}
    • AutomixIQ requires devices to have a unique identifier. To provide it, implement getDeviceIdentifier.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun getDeviceIdentifier(): String? {
        return this.deviceIdentifier
    }
    // ...
}
    • This delegate provides hooks to inject cached track audio data and metadata, to shortcut for efficiency and prevent AutomixIQ from querying audio data and metadata to your custom backend if they are already available. This also enables offline playback when all required data is available on device. To provide this information, implement syncGetTrackMetadata and syncGetCachedTrack. If no cache exists and you do not support offline playback, these functions should return null.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun syncGetTrackMetadata(trackId: String?): AutomixSDKTrackModel? {
        // If I have this track cached, I should be able to tell its metadata
        if (!this.isTrackCached(trackId)) {
            return null;
        }
        return this.metadataForTrack(trackId)
    }

    override fun syncGetCachedTrack(trackId: String): ByteArray? {
        // If I have this track cached, I should be able to decrypt it and send the data
        if (!this.isTrackCached(trackId)) {
            return null
        }
        return this.unencryptedDataForTrack(trackId)
    }
    // ...
}
    • At any time, when track metadata is available, AutomixIQ informs the delegate that track metadata is available. You can use this function to cache it for later use if you want.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun cacheMetadata(metadata: List<AutomixSDKTrackModel>) {
        this.metadataForTrack[metadata.trackId] = metadata
    }
    // ...
}
    • AutomixIQ will need you to download the trackmetadata as it needs it, by calling asyncGetTrackMetadata. Return the metadata by running onComplete, or an error by running onError.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun asyncGetTrackMetadata(trackId: String,
                                       onComplete: ((AutomixSDKTrackModel) -> Unit)?,
                                       onError: ((AutomixSDKError) -> Unit)?) -> Unit) {
        this.getTrackMetadata(trackId) { error, trackMetadata ->
            val automixSDKTrackModel = translateMetadata(trackMetadata)
            if (error != null) {
                onError(error)
                return@getTrackMetadata
            }
            onComplete(automixSDKTrackModel)
            return@getTrackMetadata
        }
    }
    // ...
}
    • When AutomixIQ needs to download a track, it will ask for the url for it by calling asyncGetStream. It supports sessionID if you want urls to expire and return 403s. Return the requested URL by running onComplete, or an error by running onError.
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun asyncGetStream(streamingDeviceId: String, sessionId: String, trackId: String, 
                                onComplete: ((String /* URL */, String? /* newSessionId */) -> Unit)?,
                                onError: ((AutomixSDKError) -> Unit)?) {
        this.getTrackURL(streamingDeviceId, sessionId, trackId) { error, url, newSessionId ->
            if (error != null) {
                onError(error)
                return@getTrackURL
            }
            onComplete(url, newSessionId, null)
        }
    }
    // ...
}
    • When a track finishes downloading, AutomixIQ passes the audio data for you to cache if you need it:
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun trackCompletedDownload(trackId: String, data: ByteArray) {
        this.cacheTrack(trackId, data)
    }
    // ...
}
    • As a track is being downloaded, AutomixIQ will report download progress:
    public class MyAutomixSDKStreamingListener: AutomixSDKStreamingListener {
    // ...
    override fun trackBufferingProgress(trackId: String, progress: Float) {
        this.updateTrackDownloadProgress(trackId, progress)
    }
    // ...
}

    Do keep in mind this download is done lazily, as AutomixIQ requires data for analysis / playback. There is no guarantee a track will download in its entirety if the track is not played in its entirety.
