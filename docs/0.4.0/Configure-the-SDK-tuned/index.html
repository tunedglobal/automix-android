    Configure the SDK with Tuned Global Backend¶

    To initialise AutomixIQ with Tuned Global's APIs and services, you need

    • A store ID and an application ID provided by the Tuned Global team
    • You need to implement the TunedRestHook interface

    Implement your TunedRestHook¶

    TunedRestHook is an interface that facilitates communication between the Tuned Global Backend and the AutomixIQ SDK regarding the user's access token and device information. Extend the TunedRestHook class and implement the required functions as shown below:

    • asyncGetAuthenticatedDevice
    • providesAuthenticatedDevice
    • uniqueDeviceIdentifier
    • tunedRestTokenExpired

    You can find a detailed explaination in API Reference

    public class MyTunedRestHook: TunedRestHook {


    // Return the deviceId from Tuned service through onComplete
    fun asyncGetAuthenticatedDevice(deviceIdentifierForVendor: String,
                                    onComplete: ((String) -> Unit)?,
                                    onError: ((AutomixSDKError) -> Unit)?) {
        val myTunedDeviceId: String = tunedDeviceId
        onComplete?.invoke(myTunedDeviceId.toString())
        return
    }

    // Tell AutomixIQ whether it needs to use the device provided (true) from uniqueDeviceIdentifier()
    // or register a device on its own (false).
    fun providesAuthenticatedDevice(): Boolean = true // or false

    // Return a UUID that is used to identify a device. This is the `deviceIdentifierForVendor`
    fun uniqueDeviceIdentifier(): String {
        val uuid: String = myUniqueDeviceIdentifier
        return uuid
    }

    // Use the info provided from the parameters to refresh the Tuned user token 
    // and return TunedAuthenticationResponse and userID through onComplete
    override fun tunedRestTokenExpired(deviceIdentifierForVendor: String,
                                        countryCode: String,
                                        tunedUserId: String?,
                                        oldAccessToken: String?,
                                        refreshToken: String?,
                                        tokenType: String?,
                                        onComplete: ((TunedAuthenticationResponse?, String) -> Unit)?,
                                        onError: ((AutomixSDKError) -> Unit)?) {
        // your implementation of refreshing Tuned user Token                                    
        refreshToken(refreshToken, tokenType, deviceIdentifierForVendor) { authToken ->
            val userId = tunedUserId
            val tunedAuthenticationResponse = TunedAuthenticationResponse(
                it.accessToken,
                "bearer",
                it.expiration.toInt(),
                it.refreshToken,
                userId
            )
            onComplete?.invoke(tunedAuthenticationResponse, userId)
        }
    }
}

    Initialise AutomixSDK instance¶

    Create an AutomixSDKConfig first. You can get the store ID, application ID from the Tuned Global team.

    val automixConfig = 
    AutomixSDKConfig.Builder()
        .country(myCountryISO2Code) // e.g. "US", "AU"
        .storeId(myTunedStoreId )
        .applicationId(myTunedApplicationId)
        .logLevel(AutomixSDKLogLevel.DEBUG)
        .build()

    Initialise the AutomixSDK instance with the TunedRestHook and AutomixSDKConfig created in previous steps. We recommend initialising the SDK once your music background service is up and running.

    Note

    AutomixIQ requires an active internet connection for its initial verification. After this initial setup, you can initialise it offline. However, note that there is a grace period of 7 days, after which an internet connection will be required again for re-verification.

    AutomixSDK.initialise(applicationContext,
                        myTunedRestHook,
                        automixConfig,
                        onComplete = { automixStatus -> 

        // The AutomixIQ public functions can be used from this point onwards :)

        // We can set a cached music provider if we want to support offline playback
        val cachedMusicProvider: AutomixSDKCachedMusicProvider = myCachedMusicProvider
        AutomixSDK.getInstance()?.setCachedMusicProvider(cachedMusicProvider)

        // We can add a status listener to get UI updates and play events, you can add as many AutomixSDKStatusListener as you like
        val automixSDKStatusListener: AutomixSDKStatusListener = myAutomixSDKStatusListener
        AutomixSDK.getInstance()?.addStatusListener(automixSDKStatusListener)

        // ...
    }, onError = { error, automixStatus -> 
        // This means you do not have permission to use the AutomixIQ or when there is no internet connection.
        return
    })

    Implement AutomixSDKCachedMusicProvider (Optional)¶

    AutomixIQ supports offline playback using local music data. You can implement the AutomixSDKCachedMusicProvider to inject cached track metadata and raw audio data, extract raw audio data downloaded by the SDK, and monitor the download progress. This approach enhances efficiency and prevents AutomixIQ from streaming audio data and downloading metadata from Tuned Global's APIs if they are already available. It also enables offline playback when all the required data is available locally.

    Implement the AutomixSDKCachedMusicProvider and add it to the SDK after initialise.

    You can find a detailed explaination in API Reference

    SetCachedMusicProvider
    AutomixSDK.getInstance()?.setCachedMusicProvider(MyCachedMusicProvider())
    MyCachedMusicProvider
    public class MyCachedMusicProvider: AutomixSDKCachedMusicProvider {

    // Provide the track metadata. For offline playback, this is required.
    override fun syncGetTrackMetadata(trackId: String?): AutomixSDKTrackModel? {
        // If the track is not cached, return null and AutomixIQ will try to fetch it.
        if (!isTrackCached(trackId)) {
            return null
        }
        return metadataForTrack(trackId)
    }

    // Provide the raw audio byte data of the audio file. For offline playback, this is required.
    override fun syncGetCachedTrack(trackId: String): ByteArray? {
        // If the track is not cached, return null and AutomixIQ will try to stream it.
        if (!isTrackCached(trackId)) {
            return null
        }

        return rawAudioDataForTrack(trackId)
    }

    // AutomixIQ will stream and download the tracks it needs to play when they are not provided from `syncGetCachedTrack`. 
    // It will do so lazily (as playback or analysis requires). 
    // When a track finished downloading, AutomixIQ will pass the raw audio byte data here for caching.
    override fun trackFinishedDownloading(trackId: String, data: ByteArray) {
        cacheTrack(trackId, data)
    }

    // Monitor the download progress
    override fun trackBufferingProgress(trackId: String, progress: Float) {
        updateTrackBufferingProgress(trackId, progress)
    }
}

    Check initialised state¶

    To check whether AutomixIQ is initialised 

    AutomixSDK.getInstance()?.initialised()

    Shut down¶

    To shut down AutomixIQ

    AutomixSDK.terminate()

    After you have terminate the SDK, you will need to reinitialise if you want to use AutomixIQ again.
